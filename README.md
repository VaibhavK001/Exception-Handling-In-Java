# 🛡️ Exception Handling in Java

A beginner-friendly collection of Java programs covering core **exception handling** concepts — from basic syntax to advanced nested and multiple catch scenarios.

---

## 📁 Project Structure

| File | Description |
|------|-------------|
| `Main.java` | Basic try-catch syntax |
| `MultipleCatchBlocks.java` | Handling multiple catch blocks |
| `Test.java` | Multiple catch blocks syntax practice |
| `TestOne.java` | try-catch-finally syntax |
| `TestTwo.java` | `finally` runs even when `return` is used in `try` |
| `TestThree.java` | Nested try-catch practice |
| `TestFour.java` | try-catch-finally flow |

---

## 📚 Concepts Covered

- ✅ **Basic try-catch** – Catching exceptions to prevent runtime crashes
- ✅ **Multiple catch blocks** – Handling different exception types separately
- ✅ **try-catch-finally** – Ensuring cleanup code always executes
- ✅ **`finally` with `return`** – Understanding `finally` executes even after a `return` in `try`
- ✅ **Nested try-catch** – Fine-grained exception handling inside another try block

---

## 💡 Key Takeaways

> Exception handling makes your Java programs **robust and crash-resistant**.

- `finally` **always** executes — whether an exception occurs or not
- Multiple `catch` blocks let you handle **specific exception types** differently
- Nested `try-catch` provides **granular control** in complex logic
- Never leave a `catch` block empty — always log or handle the error
